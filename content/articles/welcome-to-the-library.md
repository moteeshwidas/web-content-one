# Welcome to the Library

This website is rendered on the client and pulls markdown files directly from GitHub.

## What You Can Publish

- Articles under `content/articles/`
- Online books under `content/books/<bookSlug>/chapters/`
- Rich metadata in frontmatter: title, date, tags, and category

## Why This Setup Works

1. Your writing stays in plain text.
2. GitHub gives version history and review workflows.
3. The website can update as soon as markdown files change.

## Quick Author Tips

- Keep paragraphs short.
- Use headings every 3-6 paragraphs.
- Add at least one concrete example in each section.

Happy publishing.
